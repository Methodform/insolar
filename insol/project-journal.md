## 1. Что за продукт

**Инсоляр** — веб-сервис моделирования положения и движения солнца, теней и инсоляции
земельного участка (ИЖС/дача) в реальном времени. Пользователь задаёт участок,
расставляет постройки и озеленение, видит тени и инсоляцию, получает документ.

**Ядро ценности:** не «моделирование солнца», а снятие риска дорогой необратимой ошибки
(снос, спор с соседом, сад в тени) и получение уверенного, задокументированного решения
до начала стройки. Отличие от конкурентов — **живой интерактив** (двигаешь дом → тени
пересчитываются), а не статичный отчёт.

Живёт по адресу: https://methodform.github.io/insolar/ (ванильная версия — корень,
React-версия — `/app/`).

## 2. Архитектура и стек

Две параллельные версии в одном репозитории (Methodform/insolar):

- **Ванильная** (`index.html`, ~2600 строк) — single-file HTML/CSS/JS + Three.js r128 (CDN).
  Работает на GitHub Pages без сборки. Полный референс всех фич.
- **React + Radix** (`react-app/`) — Vite + React 18 + @radix-ui/themes + @radix-ui/react-icons
  + three@0.160 + maplibre-gl. Собирается в CI, деплоится в `/app/`. Идёт портирование к паритету.

Маркетинг — статические HTML: `landing.html` (лендинг, JTBD, калькулятор экономии,
переключатель лето/зима с фото), `blog.html` (блог), `article-insolyaciya.html` (первая статья).

### Структура React-приложения
- `src/App.jsx` — оболочка Radix, всё состояние и панели.
- `src/engine/astronomy.js` — чистый JS-движок (астрономия + геометрия + инсоляция), без Three.
- `src/three/Viewport.jsx` — 3D-сцена (Three.js): солнце, тени, здания, гизмо, аналитика, карта-подложка, маркеры.
- `src/three/PlanEditor.jsx` — редактор плана (вид сверху): прямоугольник/контур/дорожка, подложка-чертёж.
- `src/three/MapView.jsx` — участок на реальной карте (MapLibre + OpenFreeMap).
- `src/three/SunPath.jsx` — диаграмма пути солнца (небесный купол: сезонные дуги, стороны света, форма участка в центре, траектория текущего дня под точкой солнца).
- `src/three/HeatMap.jsx` — годовая тепловая карта (canvas). *(в UI сейчас не подключена)*
- `src/three/ZoneMap.jsx` — схема рекомендаций зонирования.
- `cadastre-proxy/worker.js` — Cloudflare Worker: участок по кадастровому номеру (НСПД).

## 3. Движок расчётов (`astronomy.js`)

- Позиция солнца — алгоритм Meeus/SunCalc (в UTC, ввод/вывод в местном поясе).
- Инсоляция — метод теневого полигона (без raycast): выпуклая оболочка footprint ∪ проекции тени.
- Нормы СанПиН 1.2.3685-21: непрерывная инсоляция ≥ 2.5/2.0/1.5 ч по климатическим зонам
  (север ≥58°, центр 48–58°, юг <48°).
- Отступы СП 53.13330 / СП 42.13330: дом 3 м, прочее 1 м; противопожарные СП 4.13130.
- Геометрия: площадь, offsetInward (отступ внутрь), plotBasis (базис участка по длинной стороне),
  rectFromDrag (прямоугольник вдоль участка), clampToPoly.
- Окна: facadeWindows + windowInsolation — только для дома/бани и нарисованных зданий с крышей.
- Контрольные точки (reportData): сетка 5×5 (N=4) в базисе участка, внутри контура.

## 4. Полный список функций

**Базовые (Free):** участок по координатам точек или по кадастровому номеру; путь солнца;
3D-тени в реальном времени; забор (светло-серый, без толщины); здания/постройки/озеленение;
инсоляция в точке и по контрольным точкам участка (зелёные/красные маркеры); длина тени;
компас С/Ю/В/З в сцене; тема светлая/тёмная/системная; участок на реальной карте (схема).

**Модели (процедурные, «+ Типовое»):** дом, баня, беседка, навес, шатёр, дерево, куст.
Быстрая посадка деревьев/кустов кликом по участку. Дорожки — в режиме «Рисовать».

**Редактор плана («Рисовать»):** прямоугольник (параллельно сторонам участка), контур с
углами 90°, дорожки (ширина), перетаскивание/поворот(R)/удаление(Del), линии отступов 1/3 м,
сетка, подложка-чертёж (загрузка картинки).

**3D-взаимодействие:** гизмо перемещения (оси вдоль сторон участка), поворот, масштаб;
перетаскивание с ограничением по нормативным отступам (объект держится в участке).

**Pro:** 3D-аналитика поверхностей («тепловизор», плавный градиент по земле/фасадам/крышам/забору,
выбор месяцев, учёт рассеянного света, легенда); нормативный отчёт по СанПиН (PDF, в футере правой
панели, «Скачать PDF отчёт»); сохранение/загрузка проекта (JSON).

**Всегда доступно в шапке:** памятка нормативных отступов (модалка), рекомендации по зонированию.

**Карта:** окно «Показать на карте» — OpenFreeMap (схема, бесплатно, без ключа); подложка-схема
в 3D — растровые тайлы MapTiler по ключу (встроенному в сборку или введённому вручную).

## 5. Карты и внешние данные

- **Схема улиц:** OpenFreeMap (векторные тайлы OSM) — бесплатно, без ключа, коммерция ок. Рендер MapLibre.
- **Схема-подложка в 3D:** растровые тайлы MapTiler streets (бесплатный тариф, ключ). Static Maps API
  НЕ используем — он платный (был баг «Invalid key» из-за него).
- **Спутник:** удалён из UI по решению (оставили только схему).
- **Кадастр:** Cloudflare Worker → НСПД `api/geoportal/v2/search/geoportal`, геометрия EPSG:3857 → WGS84.
  Риск анти-бота НСПД; ручной ввод точек — всегда рабочий fallback.
- **Ключ MapTiler:** встраивается в сборку через `VITE_MAPTILER_KEY` (GitHub secret `MAPTILER_KEY`),
  защищается ограничением по домену. Поле ручного ввода — fallback, когда встроенного ключа нет.
  Полностью скрыть клиентский ключ нельзя; при росте — кэш тайлов, платный тариф или self-host.

## 7. Стратегия (JTBD / GTM)

Подробно — в `jtbd-insolation-service.md` и `gtm-roadmap.md`.

- **Сегменты:** частник ИЖС (эмоц. джоба «где ставить, не ошибиться»), проектировщик
  (нормативный отчёт), риелтор (презентация участка).
- **Beachhead — частник ИЖС/дача:** продукт под него готов, массовый рынок, виральность, SEO.
- **Монетизация:** щедрый Free → платные подписки. Единый модал тарифов по клику на любую платную
  функцию. Линейка: **Месяц / Сезон (6 мес, рекомендуемый) / Год** плюс разовый «паспорт участка» по
  ссылке. Акцент на **Год как центр** — «живой планировщик круглый год» (посадил дом сейчас, вернулся
  весной пересадить сад). Под подпиской: объекты/постройки, выбор даты, 3D-аналитика, PDF-отчёт,
  сохранение, кадастр по номеру.
- **Позиционирование:** «не отчёт, купленный один раз, а живой участок, который можно переиграть —
  и в любой момент выгрузить паспорт или документ по СанПиН».
- **Ключевой козырь vs конкуренты (smotrisunduk и др.):** интерактив + шаринг живой ссылкой +
  нормативный СанПиН, чего у статичных «паспортов» нет.
- **Нейминг:** обсуждались варианты под .solar и standalone (Ярило, Погожий, moy.solar, sad.solar).
  Финал не зафиксирован; текущее рабочее — «Инсоляр».
- **Конкуренты:** СИТИС Солярис (нормативка, тяжёлый десктоп), Shadowmap/SunTrace3D (3D-тени, без
  СанПиН/кадастра), smotrisunduk (потребительский PDF-паспорт с розой ветров).

## 10. Известные баги и как решены
- `</script>` в шаблон-строке ломает печать → экранирование `<scr${''}ipt>`.
- Разрыв тени от забора/объектов → узкий near/far теневой камеры + меньше normalBias.
- Краш при добавлении дерева → `mesh.position` только для чтения; заменено на `.position.set()`.
- «Invalid key» на 3D-подложке → MapTiler Static Maps платный; перешли на растровые тайлы.
- «Тонкая серая линия» сквозь панель → panelBackground="solid".
- Тепловая карта «полосой» → отсечение солнца ниже 5° (длинные тени у горизонта).
- Локальная сборка React невозможна (нет node_modules) → проверка `node --check` + баланс скобок, сборка в CI.
- `clampToPoly` не был импортирован в `Viewport.jsx` (используется в `plant()`) → латентный ReferenceError при посадке дерева/куста кликом. Не проявлялся, т.к. `plantMode` всегда `null` (быстрые посадки убраны). Исправлено добавлением импорта из `astronomy.js`.
