## 11. Анализ конкурента GeoRisk (geo-risk.ru)
- **Запрос:** какие методы использует geo-risk.ru для определения границ участка.
- **Сделано:** разобран сайт + сетевой ответ (пользователь прислал JSON).
- **Вывод:** источник границ — **НСПД** (nspd.gov.ru), тот же, что у нашего `cadastre-proxy`.
  Признаки в ответе: `categoryName:"Земельные участки ЕГРН"`, `externalKey`, `interactionId`, схема
  `options`. GeoRisk **проксирует НСПД через свой бэкенд с кэшем** (поля `cacheHit`, `summary`,
  `rawProperties`). Границы не вычисляют — берут отмежёванную геометрию из ЕГРН.
- **Рекомендации для нас:** (1) добавить кэш в наш Worker (Cloudflare KV/Cache, ключ = кад. номер) —
  снимает анти-бот/лимиты НСПД; (2) проверить проекцию: ответ приходит в градусах WGS84 при лейбле
  `EPSG:3857` — убедиться, что воркер не делает двойной пересчёт (тест на `63:32:1603003:3535`).

## 15. Стратегия: игровые механики «Фермы» для сервиса
- **Запрос:** какие игровые механики игры «Ферма» можно использовать в сервисе.

  (сегмент **дачник-садовод**, ретеншен, онбординг), и **неуместны** в потоке «построить дом» / «купить

  квесты / to-do по сезону; (3) прогресс/уровни онбординга (XP-шкала до aha); (4) стройка/обустройство
  участка (декор → шкала обустроенности, «паспорт заполняется»); (5) достижения/бейджи («инсоляция по
  СанПиН ✓», «не затеняю соседа»); (6) «растёт/вянет» (грядка в тени = слабый урожай) как игровой сигнал
  поверх честного расчёта; (7) соседи/визиты → шаринг ссылкой, сравнение, тень соседа; (8) альманах
  культур (сколько солнца нужно томату/огурцу/ягоде).
- **Топ-3 внедрить первыми:** рост сада по годам (#1) → сезонные квест-карточки + напоминания (#2) →
  прогресс-онбординг до aha + бейдж «план готов / норма ✓» (#3, #5).
- **Чего НЕ брать** (удешевит инструмент, манипулятивно): энергия/таймеры, платное ускорение роста
  (pay-to-speed), лутбоксы/случайные награды, жёсткие стрики со «сгоранием прогресса».
- **Итог позиционирования:** не «превратить в игру», а взять сезонный цикл, рост во времени, наглядный
